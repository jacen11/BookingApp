# Booking App

Приложение для бронирования услуг салонов красоты, барбершопов, спа и клиник.

## Возможности

- **Поиск услуг** — поиск по категориям (салоны, барбершопы, спа, клиники)
- **Просмотр провайдеров** — детальная информация об услугах, отзывах и ценах
- **Бронирование** — выбор даты и времени, подтверждение записи
- **Оплата** — поддержка различных способов оплаты
- **Мои бронирования** — просмотр и управление активными и прошедшими записями
- **Профиль** — управление личными данными и настройками

## Архитектура

Проект построен на принципах **Clean Architecture** с использованием паттерна **MVVM**:

```
app/src/main/kotlin/dev/pastukhov/booking/
├── data/                    # Слой данных
│   ├── local/              # Локальная база данных (Room)
│   ├── remote/             # Сетевой API (Retrofit)
│   ├── repository/         # Реализации репозиториев
│   └── mock/               # Моковые данные
├── domain/                 # Доменный слой
│   ├── model/              # Доменные модели
│   ├── repository/         # Интерфейсы репозиториев
│   └── usecase/            # Бизнес-логика
├── presentation/           # Слой представления
│   ├── model/              # UI-модели и состояния
│   ├── navigation/         # Навигация
│   ├── ui/                 # UI-компоненты
│   │   ├── components/     # Переиспользуемые компоненты
│   │   └── screens/        # Экраны приложения
│   └── viewmodel/          # ViewModel-и
├── di/                     # Модули внедрения зависимостей (Hilt)
└── ui/theme/               # Тема приложения
```

## Требования

- Android SDK 28 (Android 9.0) и выше
- JDK 21
- Gradle 9.0.1

## Сборка

### Предварительные требования

1. Установите Android Studio Arctic Fox или более новую версию
2. Убедитесь, что установлен JDK 21

### Сборка debug-версии

```bash
./gradlew assembleDebug
```

### Сборка release-версии

```bash
./gradlew assembleRelease
```

### Запуск тестов

```bash
./gradlew test
```

## Экраны приложения

1. **Splash Screen** — Загрузочный экран с логотипом
2. **Login** — Экран входа в приложение
3. **Home** — Главный экран с категориями и популярными провайдерами
4. **Search** — Поиск услуг и провайдеров
5. **Provider Detail** — Детальная информация о провайдере
6. **Select Date & Time** — Выбор даты и времени записи
7. **Booking Confirmation** — Подтверждение деталей бронирования
8. **Payment** — Оплата забронированной услуги
9. **Booking Success** — Успешное бронирование
10. **My Bookings** — Список всех бронирований
11. **Profile** — Профиль пользователя и настройки

## Конфигурация

### Room Database

База данных создаётся автоматически при первом запуске. Схема сохраняется в `app/schemas/`.

### Mock Data

Приложение использует моковые данные для демонстрации функциональности. Для подключения реального API необходимо изменить реализацию репозиториев в `data/repository/`.
